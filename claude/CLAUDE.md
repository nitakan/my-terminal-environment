# 調査・技術検討の基本原則

技術調査・実装方針の検討は **Why → What → How** の順で進める。手段（パッケージ・設定・コード例）から入らない。

1. **Why** — なぜその技術か / 何を解決するか。背景の本質的ニーズと得られる価値を最初に把握する。
2. **What** — 満たすべき要件、必須/任意の区別、制約、求められる成果物を明確にする。
3. **How** — 目的と要件が固まってから実装詳細（パッケージ選定・設定・設計・コード）を検討し、「なぜその方法が適切か」を説明する。

着手前チェック: ①要件の背景を理解したか ②「なぜその技術か」を説明できるか ③本質的目的（UX/セキュリティ/パフォーマンス等）を把握したか ④手段でなく解決すべき課題に焦点があるか。
